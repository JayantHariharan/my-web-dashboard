# Contributing to PlayNexus 🚀

First off, thanks for taking the time to contribute! It's people like you that make PlayNexus better for everyone.

## 🌈 How Can I Contribute?

### Reporting Bugs
If you find a bug, please open an **Issue** and include:
- A clear, descriptive title.
- Steps to reproduce the bug.
- What you expected to happen vs what actually happened.

### Suggesting Enhancements
We're always looking for new features! When suggesting an enhancement, please explain:
- Why this feature would be useful.
- How you imagine it working.

### Pull Requests
Ready to write some code? Follow these steps:
1. **Fork** the repository to your own account.
2. **Clone** it locally: `git clone https://github.com/YourUsername/my-web-dashboard.git`
3. Create a **Feature Branch** from `develop`: `git checkout -b feature/cool-new-thing`
4. Make your changes and test them locally (see the local dev guide in README).
5. **Commit** your work with a clear message: `git commit -m "feat: add neon glow to buttons"`
6. **Push** to your fork and open a **Pull Request** targeting our `develop` branch.

## 🏗️ Project Structure & Tech Stack

- **Core:** HTML5, Vanilla CSS3 (glassmorphism), Vanilla JS.
- **Workflow:** GitHub Actions + FTP-Deploy.
- **Organization:**
    - `src/`: All live files (HTML, CSS, JS).
    - `tests/`: Smoke tests (Playwright).
